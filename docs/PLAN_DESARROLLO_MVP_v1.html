4. Detalle por fase

Fase 0 — Fundamentos y entorno (Semana 1)

Objetivo: Base técnica lista para trabajo en paralelo del equipo.

Actividades

  • Monorepo pnpm + Turborepo: apps/web, apps/desktop, packages/, supabase/
  • Proyecto Supabase (dev + staging), Next.js 14, Electron base
  • Git, ramas, deploy Vercel staging, convenciones de equipo
  • Definir formato código de barras Code 128 con contadores

Entregables

  • Repositorio funcionando
  • Supabase conectado desde web y desktop
  • Login básico en web
  • Documento arquitectura v1
  • Formato código de barras acordado

Fase 1 — Backend, modelo de datos y seguridad (Semanas 2–3)

Objetivo: BD completa, roles, permisos y API funcional.

Modelo de datos

  • Tablas: entidades, sedes, ambientes, activos, historial_cambios, profiles
  • Estados: PREREGISTRADO, REGISTRADO, DADO_DE_BAJA
  • Correlativo automático por entidad
  • Depreciación calculada (vistas SQL) + triggers de historial

Seguridad RLS

  • CONTADOR: acceso total · ADMIN_ENTIDAD: solo su entidad
  • Storage fotos/PDF con compresión (~500 KB máx.)

Criterios de aceptación

  • Contador crea activo → REGISTRADO + correlativo
  • Admin crea activo → PREREGISTRADO
  • Admin no ve otras entidades

Fase 2 — Plataforma web (Semanas 4–6)

Objetivo: Web operativa para contador y admin de entidad.

Módulo contadorMódulo admin entidad
Entidades · Jerarquía Sede/Ambiente · Inventario con filtros
Ficha activo · Usuarios · Bandeja preregistrados 		Mi inventario · Registrar activo (PREREGISTRADO)
Editar ubicación · Historial · Mis reportes

Criterios de aceptación

  • Flujo preregistro → validación contador completo
  • Admin solo edita ubicación
  • Filtros < 1 s con 2.000 activos de prueba

Fase 3 — App de escritorio Windows (Semanas 7–9)

Objetivo: Herramienta de campo: escaneo, registro, etiquetas ZPL y offline.

MóduloFuncionalidad
LoginSesión persistente Supabase Auth
Escaneo USBPistola → búsqueda instantánea → ficha
RegistroFormulario completo + foto + PDF
ZPL (ZSim)Etiquetas Code 128 → Honeywell PC42E-T
OfflineSQLite + cola sync automática al reconectar

Comportamiento offline

SituaciónComportamiento
Sin internet, primera vezNo puede hacer login
Sin internet, sesión previaTrabaja con sesión cacheada + SQLite
Vuelve internetSync automático de cola pendiente
Impresión ZPL100% offline via USB

Criterios de aceptación

  • 4 h offline sin pérdida de datos
  • Etiqueta Code 128 legible por pistola
  • Escaneo → ficha < 500 ms online
